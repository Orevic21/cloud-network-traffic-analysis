# Network Traffic Analysis Using Wireshark

## Executive Summary

This project focused on the capture and analysis of network traffic using Wireshark to investigate protocol utilization, DNS activity, encrypted communications, and TCP connection establishment. The objective was to develop practical traffic analysis skills applicable to cybersecurity monitoring, incident investigation, and cloud security operations.

A 10-minute 47-second packet capture was conducted during normal system activity. Analysis identified TCP as the dominant transport protocol, observed DNS resolution requests for external domains, confirmed the presence of encrypted TLS and QUIC communications, and verified successful TCP three-way handshakes. Additionally, VLAN-tagged traffic associated with a virtualized environment was detected.

# 2. Methodology

Network traffic was captured during routine browsing and system activity. Analysis was performed using multiple Wireshark features including:

* Protocol Hierarchy Statistics
* Endpoints Analysis
* DNS Filtering
* TCP Stream Analysis
* TLS Inspection

The following display filters were utilized:

```text
dns
tcp
tls
```

The objective was to identify communication patterns, protocol usage, encryption adoption, and network behaviors relevant to security monitoring operations.

## 3.2 Notable Protocol Observations

### Malformed Packet

One malformed TCP packet was identified during analysis.

Potential causes include:

* Packet corruption
* Capture anomalies
* Protocol decoding issues
* Transmission errors

No evidence suggested malicious activity associated with the malformed packet.

### Legacy Discovery Protocols

The following protocols were observed:

* NetBIOS
* SMB
* SSDP

These protocols are commonly associated with:

* Device discovery
* File sharing
* Local network communications

### VLAN Tagged Traffic

802.1Q VLAN traffic accounted for approximately 51.8% of observed packets.

Analysis indicates this behavior was generated by VirtualBox virtual networking components and is consistent with expected virtualization activity rather than abnormal network behavior.

# 6. TCP Connection Analysis

## TCP Three-Way Handshake Verification

A successful TCP connection establishment sequence was identified.

| Frame | Type     |
| ----- | -------- |
| 375   | SYN      |
| 377   | SYN, ACK |
| 379   | ACK      |

### Handshake Process

1. Client initiated connection using SYN.
2. Server acknowledged request using SYN-ACK.
3. Client completed negotiation using ACK.
